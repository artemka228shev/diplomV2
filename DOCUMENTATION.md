# Техническая документация Habitify

## Архитектура приложения

### MVC паттерн

Приложение построено по паттерну Model-View-Controller:

**Models** - работают с базой данных, инкапсулируют бизнес-логику сущностей
**Views** - HTML-шаблоны с встроенным PHP для вывода данных
**Controllers** - обрабатывают запросы, взаимодействуют с моделями, возвращают представления или JSON

### Кастомный роутер

Файл: `app/Core/Router.php`

Основные возможности:
- Поддержка HTTP методов GET, POST, PUT, DELETE
- Параметры URL через `{param}` синтаксис
- Автоматическое преобразование в регулярные выражения
- Вызов контроллеров по методу или closure

Пример маршрута:
```php
$router->get('/habits/{id}', [HabitController::class, 'show']);
```

### База данных

Таблицы:
- `users` - пользователи с ролями и подписками
- `habits` - привязаны к пользователю, типы boolean/quantitative
- `habit_logs` - записи выполнения с датой и оценкой
- `reminders` - напоминания (планируется)
- `subscriptions` - история подписок
- `ads_clicks` - аналитика рекламы

Индексы созданы на часто используемых полях (email, user_id, date).

## Безопасность

### Пароли
Хеширование через `password_hash()` с алгоритмом PASSWORD_DEFAULT (bcrypt).

### SQL инъекции
Используются подготовленные выражения PDO во всех запросах.

### XSS защита
Все выводимые данные экранируются через `htmlspecialchars()`.

## Система подписок

### Типы подписок

| Тариф | Привычки | Реклама | Статистика |
|-------|----------|---------|------------|
| free | 5 | Да | Базовая |
| basic | 15 | Нет | Расширенная |
| premium | Безлимит | Нет | Расширенная + экспорт |

### Проверка лимитов

В `HabitController::create()` проверяется количество привычек перед созданием:
```php
$habitCount = $this->auth->getHabitCount();
$limit = $this->auth->getHabitLimit();

if ($habitCount >= $limit) {
    // Возврат ошибки с предложением улучшить тариф
}
```

### Админ-панель

Функционал:
- Просмотр всех пользователей
- Изменение подписки (select)
- Бан/разбан
- Назначение прав администратора

Доступ только для пользователей с `role = 'admin'`.

## API Endpoints

### Аутентификация
- `POST /login` - вход, возвращает JSON с redirect
- `POST /register` - регистрация
- `GET /logout` - выход

### Привычки
- `GET /habits` - список
- `POST /habits` - создание
- `PUT /habits/{id}` - обновление
- `DELETE /habits/{id}` - удаление
- `POST /habits/{id}/log` - отметка выполнения

### Админ
- `PUT /admin/users/{id}/subscription` - смена подписки
- `POST /admin/users/{id}/toggle-ban` - бан/разбан

## Статистика и аналитика

### Метрики

1. **Процент выполнения** - отношение выполненных к общему числу
2. **Средняя серия** - среднее значение streak по всем привычкам
3. **Тепловая карта** - визуализация активности за 30 дней

### Расчет streak

Серия - количество последовательных дней с выполнениями. Расчёт в `HabitLog::getStreak()`:
- Берутся последние 100 записей
- Сравниваются с текущей датой по дням
- Прерывается при пропуске дня

## Развёртывание

### Требования
- PHP 8.0+
- MySQL 8.0+
- Composer
- Веб-сервер с mod_rewrite

```
DB_HOST=localhost
DB_PORT=3306
DB_NAME=habitify
DB_USER=root
DB_PASS=
```

```bash
composer install
php -S localhost:8000 -t public
```

- Включить error_reporting(E_ALL)
- Настроить логирование
- Использовать HTTPS
- Регулярные бэкапы БД
